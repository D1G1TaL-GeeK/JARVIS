# JARVIS

Учебный старт проекта голосового помощника `JARVIS` на Rust.

Сейчас реализован самый первый этап:

- без GUI;
- без сети;
- без облачных моделей;
- с очень подробной структурой кода;
- с текстовым интерфейсом вместо микрофона, чтобы сначала спокойно понять архитектуру Rust-проекта.

Почему начинаем именно так:

- ты просил идти с самого начала;
- Rust у тебя новый, значит лучше сначала понять каркас, типы, `enum`, `struct`, `trait` и поток выполнения;
- голос, локальные модели и исполнение системных команд мы будем добавлять поэтапно, а не сваливать все в один большой и хрупкий файл.

## Что уже умеет текущая версия

- "слушать" через консольный ввод;
- записывать короткий WAV-фрагмент с микрофона по команде;
- "обрабатывать" через простой rule-based brain;
- "генерировать" ответ;
- "предлагать" безопасные команды;
- "исполнять" локальные действия;
- "информировать" пользователя через консольный вывод.

## Быстрый запуск

### Рекомендуемый запуск на Windows

Для Windows лучше запускать через:

```cmd
run-jarvis.cmd
```

Этот launcher просто находит `cargo` и запускает `cargo run` из корня проекта.

Если вместо русских букв видишь `????`, проблема обычно в старой Windows-консоли и ее шрифте. Для классического `Windows PowerShell` надежнее использовать `Lucida Console`; `Consolas` тоже может работать, но на некоторых системах ведет себя нестабильно.

### Обычный запуск

После установки Rust открой новый терминал и выполни:

```powershell
cargo run
```

Если `cargo` не найден, просто перезапусти терминал, чтобы обновился `PATH`.

`Ctrl+C` теперь обрабатывается как graceful shutdown:

- JARVIS не должен падать аварийно;
- временные записи из `recordings` должны удаляться через тот же shutdown-cleanup путь.

## Команды для первой версии

- `помощь`
- `который час`
- `покажи файлы`
- `слушай 5`
- `запиши 5`
- `прослушай запись`
- `проиграй запись`
- `повтори привет`
- `предложи команду`
- `выход`

## Что делает `слушай 5`

Команда:

```text
слушай 5
```

означает:

- записать 5 секунд с микрофона;
- сохранить результат в папку `recordings`;
- вывести путь к WAV-файлу и параметры записи.

Это еще не распознавание речи. На этом шаге мы строим только честный слой захвата звука.

Важно:

- записи сейчас считаются временными debug-файлами;
- при завершении JARVIS папка `recordings` очищается автоматически.

## Что делает `прослушай запись`

Команда:

```text
прослушай запись
```

означает:

- взять последнюю запись, которую сделал JARVIS в текущем запуске;
- воспроизвести ее как временный debug-инструмент;
- убедиться, что микрофон действительно пишет то, что ты сказал.

После выхода JARVIS эти временные записи удаляются автоматически.
